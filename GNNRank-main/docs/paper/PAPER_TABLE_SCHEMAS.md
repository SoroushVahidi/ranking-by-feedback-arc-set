# Paper Table Schemas (Canonical Outputs)

All files below are generated by `scripts/paper/rebuild_experiment_tables.py`.

## `outputs/audits/canonical_dataset_inventory.csv`
- Purpose: canonical dataset list used for paper denominator policy.
- Columns: `dataset`, `family`, `n_nodes`, `m_edges`, `adj_artifact`, `notes`.
- Sources: `paper_csv/leaderboard_per_method.csv`, `data/**/adj.npz`, `data/_AUTO/**/adj.npz`.

## `outputs/audits/canonical_dataset_inventory_summary.json`
- Purpose: machine-readable count/family-range summary.
- Key fields: `canonical_dataset_count`, `families`, `dataset_count_policy`, `special_cases`.

## `outputs/audits/method_name_canonicalization.json`
- Purpose: canonical method naming map and observed canonical method list.
- Key fields: `canonical_aliases`, `observed_canonical_methods`, `policy`.

## `outputs/paper_tables/table4_full_suite.csv`
- Purpose: full-suite per-method/config summary table.
- Columns: `method`, `config`, `method_family`, upset medians/means, `coverage`.
- Sources: `paper_csv/results_from_result_arrays.csv` (`which=upset`) filtered to canonical suite.

## `outputs/paper_tables/table5_compute_matched.csv`
- Purpose: compute-matched subset of table4.
- Filter: `runtime_sec_mean <= 1800`.
- Columns: same as table4.

## `outputs/paper_tables/table6_missingness.csv`
- Purpose: per method/config runtime/coverage/missingness and finance status.
- Columns: `method`, `config`, `valid_metrics`, `valid_runtime`, `timeouts_or_missing_runtime`, `finance_status`.

## `outputs/paper_tables/table7_best_in_suite.csv`
- Purpose: per-dataset best OURS/classical/GNN upset inputs for best-in-suite claims.
- Columns: `dataset`, `best_ours_method`, `best_ours_upset_simple`, `best_classical_method`, `best_classical_upset_simple`, `best_gnn_method`, `best_gnn_upset_simple`.

## `outputs/paper_tables/table8_runtime_tradeoff.csv`
- Purpose: per-dataset runtime/speedup/Pareto inputs.
- Columns: `dataset`, `ours_method`, `ours_runtime_sec`, `baseline_method`, `baseline_runtime_sec`, `speedup_baseline_over_ours`, `ours_not_dominated`.

## `outputs/paper_tables/paper_claims_master.json`
- Purpose: central claim-level aggregates and finance-by-variant statuses.

## `outputs/paper_tables/claim_traceability.json`
- Purpose: mapping from major claim categories to canonical files, keys, upstream sources, and generator script.

## `outputs/paper_tables/provenance_manifest.json`
- Purpose: generation metadata (script, commit, timestamp, upstream sources, policies, output list).

## Caveats
- Canonical dataset count policy is 81.
- There is no canonical 80-dataset subset for paper claims.
- Finance timeout/completion must be interpreted per method+config.
